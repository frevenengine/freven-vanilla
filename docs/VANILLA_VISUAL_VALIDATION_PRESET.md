# Vanilla Visual Validation Preset

The rc10 Vanilla visual validation preset is a small launchable scene for checking
the first data-driven Vanilla visual content pack.

It is intentionally separate from the default Vanilla experience. The default
experience stays `freven.vanilla` with the simple flat worldgen provider. The
validation scene is selected through a stack preset:

~~~text
freven.vanilla.visual_validation
~~~

## What it validates

The origin column of the preset contains:

- flat textured terrain;
- visible swatches for stone, dirt, grass, coarse dirt, and glass;
- a transparent glass wall in front of an opaque marker;
- large greedy-mesh UV probes for top faces and vertical faces;
- simple face-lighting probes for top/side visual contrast;
- a small stone alcove/canopy for occlusion and lighting contrast;
- sparse single-block markers that make material/key drift obvious in screenshots.

The scene is generated by Vanilla's own worldgen provider:

~~~text
freven.vanilla:visual_validation
~~~

The engine does not know what "grass", "stone", "glass", or "coarse dirt" means.
The provider resolves Vanilla block ids by stable block keys and emits normal
worldgen writes.

The preset intentionally relies on the engine-owned voxel shader. Vanilla must not
ship a stale `shaders/voxel.wgsl` override because that file is renderer ABI, not
Vanilla content style.

## Launch

From a DevKit distribution or development checkout that includes this Vanilla
revision:

~~~bash
freven_boot serve --instance instances/server --experience freven.vanilla.visual_validation
freven_boot play --instance instances/player --experience freven.vanilla.visual_validation -- --connect 127.0.0.1:12806 --username visual-check --devtools
~~~

For a local development build, use the appropriate `./freven_boot` path.

## Pre-launch checks

Run these before taking release screenshots:

~~~bash
freven_boot providers explain --instance instances/server --experience freven.vanilla.visual_validation
freven_boot content-assets check --instance instances/server --experience freven.vanilla.visual_validation
freven_boot content-assets inspect --instance instances/server --experience freven.vanilla.visual_validation
~~~

Expected provider default:

~~~text
worldgen = freven.vanilla:visual_validation
~~~

Expected major visual content:

- `freven.vanilla:block/stone`
- `freven.vanilla:block/dirt`
- `freven.vanilla:block/grass`
- `freven.vanilla:block/coarse_dirt`
- `freven.vanilla:block/glass`
- `freven.vanilla:textures/stone`
- `freven.vanilla:textures/dirt`
- `freven.vanilla:textures/grass`
- `freven.vanilla:textures/coarse_dirt`
- `freven.vanilla:textures/glass`

## Current rc10 coverage

This preset is expected to exercise the current landed Vanilla visual path:

- authored texture files loaded through the visual asset pipeline;
- material keys resolved into renderer-internal slots;
- fallback debug tints staying as fallback instead of the happy path;
- opaque and transparent render layers;
- transparent glass over opaque geometry;
- greedy-meshed large faces with tiled texture UVs;
- simple client-side face-lighting/tint attributes.

## Not covered by this preset yet

These are intentionally not claimed by `freven.vanilla.visual_validation` until
they are source-of-truth in the Vanilla/runtime/tooling bridge:

- canonical data-driven block visual files selected from Vanilla content;
- per-face Vanilla block visuals such as grass top/side/bottom;
- authored model files for framed glass, panes, slabs, or non-cube geometry;
- biome/color-map tint sources wired through Vanilla content;
- content family expansion for colored glass or generated soil/grass variants;
- full scene lighting, block light, ambient occlusion, shadows, or PBR.

## Regression checklist

Before an rc10 release, capture at least one screenshot from the scene and check:

- terrain uses authored textures, not only debug colors;
- glass is transparent enough to see the marker behind it;
- opaque stone remains opaque;
- grass/dirt/coarse dirt are visually distinguishable;
- large floor/wall probes tile textures instead of stretching them;
- glass stays transparent and does not corrupt opaque material sampling;
- top and side faces show the simple rc10 face-lighting contrast;
- the alcove/canopy creates obvious lighting or occlusion contrast when supported;
- missing-asset fallback colors are not visible in the happy path.

## Boundaries

This preset is Vanilla/dev/test content. It is not an engine fixture and not a
runtime save-state format.

Future visual features can extend the scene with more authored Vanilla blocks,
for example leaves, emissive blocks, framed glass, per-face grass, or colored
glass, once those features are source-of-truth in the runtime/tooling stack.
